Fluxo de uma mensagem sua

  1. Você manda mensagem no Telegram bot
  2. Telegram POST HTTPS → webhook.jaque.<dom>
  3. Cloudflare valida IP do Telegram CIDR allow-list
  4. Tunnel outbound entrega no Gateway OpenClaw :57017
  5. Gateway valida user_id contra whitelist + secret_token HMAC
  6. Roteia pra Jaque-mestre (workspace privilegiado)
  7. Jaque planeja, consulta memória, decide delegar ou agir
  8. Se delega → subagent isolado em workspace quarentena
  9. Tool call passa por canary check + audit log JSONL
  10. Resposta volta pelo mesmo caminho · custo registrado

Por que dual-LLM e quarentena

Workshop diz "Jaque NUNCA fala direto com cliente". Tecnicamente isso vira dual-LLM pattern:

  • ▸ Jaque-mestre = workspace privilegiado, vê segredos, conhece projetos, tem memória do Igor.
  • ▸ Subagents = workspaces de quarentena. Não acessam segredos do mestre. Recebem tarefa, retornam resultado.
  • ▸ Se prompt injection roubar contexto do subagent → não vaza segredo do mestre.
  • ▸ Canary token detecta tentativa de exfiltração.

04 · Decisões já tomadas por mim

Cinco escolhas que não precisam te ocupar.

Cada uma com fonte da research, alternativas consideradas, e o porquê. Se discordar, marca aqui pra eu reabrir antes do go-live.

D-T1 Topologia · 1 Jaque-mestre + 3 subagents fixos workshop + research convergem

Por quê: Workshop [01:53:23] e [04:58:56] é taxativo — "Jaque NUNCA fala direto com cliente, sempre via subagents". Research 14 valida via dual-LLM pattern. Mono-agente expõe segredos a prompt injection. Sob-demanda é caro pra MVP.

Alternativas: mono (rejeitado: sem isolamento), sob-demanda dinâmica (rejeitado: complexidade MVP), 4+ subagents (futuro pós-MVP).

GANHA

Isolamento de segredos · resistência a prompt injection · escala 0–N subagents depois.

CUSTA

Mais complexo que mono · roteamento extra (overhead ~50ms/chamada).

D-T2 Memória · Active Memory plugin nativo (Mem0 só fase 2) research 12 · custo zero extra

Por quê: Reviewer encontrou redundância V1 — Mem0 + Active Memory faziam mesmo trabalho. Plugin nativo já cobre semantic recall + injection automática. Mem0 self-hosted custaria ~US$25-45/mês de embeddings + container Postgres+pgvector extra. Não justifica fase 1.

Quando migrar: se após 1 mês de uso o recall ficar fraco (esquecimentos críticos, contradições) → migrar pra Mem0 self-hosted. Plano B pronto em research 12.

D-T3 LLM primário · DeepSeek V3.2 (não Plus OAuth) research 11 + 16 · custo R$ 8-25/mês

Por quê: V1 sugeria GPT-5.5 via Plus OAuth (US$ 20 fixo). Red-team encontrou: Plus tem teto 100-150 msg/5h. Heartbeat 20min × 16h ativas = 48 ciclos/dia + 3 subagents + você livre → estoura cota em 24-48h. Pior: Plus em loop autônomo VPS = padrão robótico → risco ban TOS → perde Codex CLI no seu laptop também.

DeepSeek V3.2: $0.252/M input + $0.378/M output. Cenário típico Jaque (~1.5M in + 0.3M out por mês) = US$ 0.49/mês ≈ R$ 2,70. Tool calling estruturado · janela 131K · PT-BR excelente. Provider independente (não Anthropic, não OpenAI).

Plus OAuth via /codex: permanece como opção MANUAL. Você manda /codex pergunta, Jaque encaminha pro Plus uma vez. Nunca em loop. Decisão D1 abaixo se vale a pena ter.

D-T4 Imagem OpenClaw · pin :2026.5.17 (não :latest) research 10 · estabilidade

Por quê: Research 10 documentou cadência agressiva de releases OpenClaw (várias por semana) com 5 breaking changes em fev–mai/2026. Versão 2026.5.5 quebrou rotas Codex acidentalmente — revertida em 2026.5.6. Usar :latest = roleta russa.

Estratégia: pin imutável + watchdog que notifica quando GHCR publica nova tag. Antes de atualizar: staging local no Windows (Docker Desktop) + 24h soak time + smoke test → só então toca VPS.

D-T5 Cloudflare · 2 Apps separadas (webhook IP-allow + admin email+2FA) research 14 · UX correta

Por quê: Reviewer pegou: V1 propunha 1 Cloudflare Tunnel só com policy "IP-allow Telegram CIDRs". Mas o mesmo tunnel também exporia Netdata e Uptime Kuma — você seria barrado por não ser do IP do Telegram.

Solução:

  • ▸ webhook.jaque.<dom> → policy IP-allow Telegram CIDRs.
  • ▸ admin.jaque.<dom> → policy email + 2FA.

13 · Glossário

Vocabulário pra acompanhar tudo isso.

OpenClaw

Framework agente self-hosted TypeScript/Node criado por Peter Steinberger. Gateway multicanal com memória, heartbeats e MCP.

MCP

Model Context Protocol. Padrão Anthropic pra agentes consumirem ferramentas externas.

DeepSeek V3.2

LLM open-weight chinês (MoE 671B/37B). Top custo×qualidade PT-BR.

OpenRouter

Gateway de LLM passthrough. 1 API key acessa 100+ modelos.

RAG

Retrieval-Augmented Generation. Agente busca info relevante antes de responder.

RPO / RTO

Recovery Point Objective e Recovery Time Objective. Jaque: RPO 24h, RTO 2h.

SOPS + age

Secrets cifrados com chave age, commitáveis em git. Decrypt só em memória no boot.

Canary token

String falsa em segredos. Se aparecer em output = tentativa de exfiltração detectada.

Heartbeat

Loop autônomo periódico. Jaque "acorda" de X em X minutos pra checar tarefas.

Dual-LLM

Pattern Simon Willison: 1 modelo privilegiado + 1 quarentena. Defesa contra prompt injection.

grammY

Framework moderno Telegram Bot em TypeScript. OpenClaw usa nativo.

Cloudflare Tunnel

Conexão outbound da VPS pra CF. Expõe serviços SEM abrir porta. Free.

restic

Backup CLI moderno. Incremental, deduplicado, cifrado cliente.

Mem0

Camada memória semântica. Postgres+pgvector. LongMemEval 94.4.

biMEDI

Brand/Market/Audit/Decision. Framework Denderson para SDR.

Circuit breaker

Limite hard: bate US$5/dia ou US$0.50/min → Jaque pausa.

eval-harness

Suíte 20-50 perguntas golden. Skill nova passa ≥80%.

3-2-1 backup

3 cópias, 2 mídias, 1 offsite. Padrão indústria.
